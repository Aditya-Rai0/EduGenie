## Local Setup

### Prerequisites

- Python 3.12+
- PostgreSQL 16 (or Supabase project)
- Redis (optional for local dev)

### Steps

```bash
# 1. Navigate to backend
cd backend

# 2. Create virtual environment
python -m venv .venv

# 3. Activate it
# Windows:
.venv\Scripts\activate
# macOS / Linux:
source .venv/bin/activate

# 4. Install dependencies
pip install -r requirements/dev.txt

# 5. Configure environment
cp ../.env.example .env
# Edit .env with your local values

# 6. Run database migrations
alembic upgrade head

# 7. Start the server
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

The API will be available at **http://127.0.0.1:8000**.

Interactive docs (auto-generated by FastAPI) at **http://127.0.0.1:8000/docs**.

## Gen AI Components

### Agent Pipeline

The AI pipeline consists of 7 specialized agents coordinated by an `OrchestratorAgent`:

| Agent | Responsibility |
|-------|---------------|
| **Intelligence Agent** | Market research, competitor analysis, topic validation |
| **Architect Agent** | Curriculum design using Bloom's taxonomy |
| **Scriptwriter Agent** | Full lesson script generation with [VERIFY] flags |
| **MediaForge Agent** | Slide decks, voice narration, video rendering, captions |
| **Evaluator Agent** | Quiz generation, capstone project briefs, flashcards |
| **Launchpad Agent** | Sales page HTML, email sequences, social media posts |
| **Optimizer Agent** | Post-launch analytics, drop-off analysis, improvement reports |

### Base Agent Architecture

All agents extend `BaseAgent` (defined in `app/agents/base.py`):

```python
class BaseAgent(ABC):
    async def run(self, input_data: AgentInput) -> AgentOutput:
        ...
```

The `OrchestratorAgent` registers agents by name and executes them in sequence, passing state between stages.

### Integrations

| Integration | File | Purpose |

| **OpenAI** | `app/integrations/openai.py` | GPT-4o text generation, embeddings, TTS |

## Configuration

All configuration is managed via Pydantic Settings in `app/config.py`. Environment variables are loaded from `.env` in the project root. Key settings groups:

- **`SUPABASE_*`** — Database connection and auth
- **`OPENAI_*`** — AI model access
- **`STRIPE_*`** — Payment processing
- **`REDIS_URL`** — Queue and cache backend
- **`DATABASE_URL`** — PostgreSQL async connection string
